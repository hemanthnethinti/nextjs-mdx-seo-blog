# ModernBlog — Next.js SEO Blog Platform

ModernBlog is a production-ready blog platform built with Next.js (Pages Router), TypeScript, Tailwind CSS, and MDX.

The project focuses on static site generation, SEO best practices, Docker-based deployment, and clean component-driven architecture.

All content is stored as local `.mdx` files and rendered at build time using SSG for fast page loads and search engine visibility.

## Live Project

https://nextjs-mdx-seo-blog.vercel.app

## Quick Start

Run the app immediately with Docker:

```bash
docker compose up --build
```

App: `http://localhost:3000`

Stop:

```bash
docker compose down
```

Useful endpoints:

- `http://localhost:3000`
- `http://localhost:3000/sitemap.xml`
- `http://localhost:3000/rss.xml`

## Why I Built This

I wanted to implement a fully static blog architecture that:

- does not depend on a database
- generates SEO-friendly HTML at build time
- supports rich MDX content
- can be deployed in a production-ready Docker setup

This project is designed to reflect real-world blog platform architecture rather than a simple tutorial app.

## Architecture Overview

### 1) Static Site Generation

All pages are generated at build time:

- Homepage
- Blog listing pages
- Paginated routes (`/blog/[page]`)
- Individual posts (`/posts/[slug]`)

This gives fast first paint, no runtime data fetching for content, and CDN-friendly output.

### 2) MDX Content Pipeline

Posts are stored in `/posts` as `.mdx` files.

At build time:

- `gray-matter` parses frontmatter
- MDX content is serialized and rendered through `next-mdx-remote`
- metadata is used for listing, pagination, and SEO
- sitemap and RSS are generated by scripts in `/scripts`

No CMS and no database are required for publishing.

### 3) Component Structure

The UI follows a simple, reusable component hierarchy:

- `Layout` wraps all pages
- `Navbar` + `Footer`
- `PostCard` for post previews
- `Pagination` for list pages
- `ThemeToggle` for theme switching
- custom MDX renderers for code blocks and images

Responsibilities are separated into `pages`, `components`, `lib`, and `types`.

### 4) Docker Setup

The project uses a multi-stage Dockerfile:

- Stage 1: install/build Next.js app
- Stage 2: copy only production artifacts
- run as non-root user
- healthcheck enabled
- Alpine base image for smaller footprint

## Writing a Post

Create a file inside `/posts`:

```mdx
---
title: "My Article"
date: "2024-01-31"
author: "Your Name"
tags: ["Next.js", "SEO"]
excerpt: "Short summary for SEO and previews."
---

# My Article

Content written in MDX.
```

After rebuilding, the post is automatically:

- listed in blog pages
- included in pagination
- added to `sitemap.xml`
- added to `rss.xml`

## SEO Implementation

Each page includes:

- title and meta description
- Open Graph metadata
- Twitter card tags

Sitemap and RSS are generated during `npm run build`.
